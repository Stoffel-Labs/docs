# Ecosystem Overview

Stoffel 0.1.0 is organized around [`Stoffel-Labs/StoffelVM`](https://github.com/Stoffel-Labs/StoffelVM), the primary repository for the CLI, compiler, Rust SDK, VM runtime, and shared bytecode/type definitions.

The product promise is developer-accessible MPC: write the computation, mark sensitive values explicitly, run the same workflow locally, and move from clear tests to secret-shared execution without becoming a protocol implementer.

## 0.1.0 repository status

| Repository | Role in 0.1.0 | Developer guidance |
| --- | --- | --- |
| `Stoffel-Labs/StoffelVM` | Primary repository for CLI, compiler, Rust SDK, VM, bytecode, runtime helper, C FFI, and examples. | Start here. Build from source for 0.1.0. |
| `Stoffel-Labs/mpc-protocols` | MPC protocol dependency used by the VM/SDK. | Pulled by Cargo as a git dependency; app developers normally use the CLI and SDK. |
| `Stoffel-Labs/stoffel-networking` | QUIC/networking dependency used by local/networked execution paths. | Pulled by Cargo as a git dependency. |
| `Stoffel-Labs/stoffel-mpc-coordinator` | Coordinator dependency used by SDK/on-chain coordination surfaces. | Pulled by Cargo as a git dependency. |

## Primary developer workflow

### 1. Install the tools

```bash
git clone https://github.com/Stoffel-Labs/StoffelVM.git
cd StoffelVM
git checkout v0.1.0
cargo build
cargo install --path crates/stoffel-cli
cargo build -p stoffel-vm --bin stoffel-run
```

For Stoffel 0.1.0, `stoffel-run` is the local MPC helper used by SDK and CLI local execution paths.

### 2. Create a project

```bash
stoffel init hello-mpc
cd hello-mpc
```

The default template currently creates a hybrid Rust/Stoffel project:

```text
hello-mpc/
в”ңв”Җв”Җ Cargo.toml
в”ңв”Җв”Җ README.md
в”ңв”Җв”Җ Stoffel.toml
в””в”Җв”Җ src/
    в”ңв”Җв”Җ main.rs
    в”ңв”Җв”Җ main.stfl
    в””в”Җв”Җ stoffel_bindings.rs
```

### 3. Compile, inspect, and run

```bash
stoffel check
stoffel build
stoffel compile --disassemble target/debug/hello-mpc.stflb
stoffel run
```

For local MPC development:

```bash
stoffel dev \
  --runner /path/to/StoffelVM/target/debug/stoffel-run \
  --parties 5 \
  --threshold 1 \
  --once

```

Use `stoffel dev --once` for a script-friendly one-shot run.

## Component details

### Stoffel CLI

The `stoffel` binary is a Cargo-like command-line tool. Current commands are:

- `init` / `new`: create projects from templates.
- `check`: validate source and MPC settings without writing bytecode.
- `compile`: compile source or projects to `.stflb`, or disassemble bytecode.
- `build`: write project bytecode under `target/`.
- `run`: run a project, `.stfl` source file, or `.stflb` bytecode file.
- `dev`: watch and rerun local MPC flows.
- `test`: run no-argument Stoffel test functions.
- `status` / `doctor`: show project health and environment status.
- `clean`: remove generated artifacts.
- `update` / `upgrade`: check or update CLI/project dependencies.

### Stoffel Lang compiler

Stoffel Lang is the source language for the VM. The 0.1.0 syntax is Python-style:

```stoffel
def main(a: int64, b: int64) -> int64:
  return a + b
```

Secret inputs are explicit:

```stoffel
def main(a: secret int64, b: secret int64) -> secret int64:
  return a + b
```

### Rust SDK

The Rust SDK is the primary application API for 0.1.0. It exposes `Stoffel::compile`, `Stoffel::load`, `with_inputs`, `with_client_input`, `execute_clear`, `execute_local`, bytecode save/load helpers, network deployment builders, and observability helpers.

```rust
use stoffel::prelude::*;

fn main() -> stoffel::Result<()> {
    let result = Stoffel::compile("def main(a: int64, b: int64) -> int64:\n  return a + b")?
        .with_inputs(&[("a", 42_i64), ("b", 58_i64)])
        .execute_clear()?;

    println!("Result: {}", result[0]);
    Ok(())
}
```

### Stoffel VM

The VM is a register-based runtime with clear values, secret shares, objects, arrays, closures, FFI support, VM hooks, bytecode support, and MPC-focused builtins such as `Share.*`, `Mpc.*`, `ClientStore.*`, and `MpcOutput.*`.

### MPC/network backends

HoneyBadger MPC, AVSS, reliable broadcast, and QUIC networking integrations are wired into the repository through Rust dependencies. Local developer workflows normally use the SDK/CLI and `stoffel-run` rather than integrating those crates directly.
