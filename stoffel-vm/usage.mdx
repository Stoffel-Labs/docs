---

Most developers should use `stoffel run`, `stoffel dev`, or the Rust SDK. The CLI includes the local development workflow needed to run MPC on your machine.

## Compile bytecode

The current bytecode extension is `.stflb`:

```bash
stoffel compile src/main.stfl --output target/debug/main.stflb
stoffel compile --disassemble target/debug/main.stflb
```

For project workflows, prefer:

```bash
stoffel build
stoffel run
```

## Run through the CLI

```bash
stoffel run target/debug/main.stflb --entry main
stoffel run src/main.stfl --input a=40 --input b=2
stoffel run --program-info
```

`stoffel run` defaults to local execution unless `--network` or `--config` is provided:

```bash
stoffel run \
  --parties 5 \
  --threshold 1
```

For `ClientStore.take_share` programs, provide client-slot inputs:

```bash
stoffel run \
  --local \
  --client-input 0=42 \
  --expected-output-clients 1 \
  --parties 5 \
  --threshold 1
```

## Watch mode

```bash
stoffel dev \
  --parties 5 \
  --threshold 1
```

Use one-shot mode for scripts:

```bash
stoffel dev --once
```

## Run through the Rust SDK

Clear local execution:

```rust
use stoffel::prelude::*;

fn main() -> stoffel::Result<()> {
    let result = Stoffel::compile("def main(a: int64, b: int64) -> int64:\n  return a + b")?
        .with_inputs(&[("a", 40_i64), ("b", 2_i64)])
        .execute_clear()?;

    println!("Result: {}", result[0]);
    Ok(())
}
```

Local MPC execution:

```rust
use stoffel::prelude::*;

# async fn example() -> stoffel::Result<()> {
let result = Stoffel::compile(
    "def main() -> int64:\n  var share = ClientStore.take_share(0, 0)\n  return share.open()",
)?
.parties(5)
.threshold(1)
.with_client_input(0, &[42_i64])
.execute_local()
.await?;
# Ok(())
# }
```

## Debugging

Useful CLI-level debugging commands:

```bash
stoffel check --print-ir
stoffel run --program-info
stoffel compile --disassemble target/debug/main.stflb
stoffel status --verbose
```
