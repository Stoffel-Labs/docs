Stoffel VM registers ordinary runtime builtins and MPC-focused module-style builtins. This page reflects the 0.1.0 release in the `stoffel` repository.

## General runtime builtins

| Builtin | Purpose |
| --- | --- |
| `print(value)` | Print a value during VM execution. |
| `type(value)` | Return a runtime type name. |
| `to_string(value)` | Convert a value to its string form. |
| `create_object(...)` | Create an object value. |
| `create_array(...)` | Create an array/list value. |
| `get_field(object, key)` | Read an object field or array/list index. |
| `set_field(object, key, value)` | Write an object field or array/list index. |
| `array_length(array)` | Return array/list length. |
| `array_push(array, value, ...)` | Append one or more values. |
| `slice(value, start, end)` | Slice arrays/strings; negative bounds are supported. |
| `contains(container, value)` | Membership test backing the `in` operator. |
| `assert(condition)` | Abort execution if a condition is false. |
| `create_closure`, `call_closure` | Closure creation and invocation support. |
| `get_upvalue`, `set_upvalue` | Captured-variable access for closures. |

In StoffelLang, prefer method syntax where the language exposes it:

```stoffel
def main() -> int64:
  var values: list[int64] = []
  values.append(10)
  values.append(20)
  return values.len()
```

## Share builtins

`Share` values represent MPC secret shares. The VM registers these names on the 0.1.0 release:

```text

```

Most share functions are also available as methods in StoffelLang examples:

```stoffel
def compute(a: Share, b: Share) -> int64:
  var sum = a.add(b)
  var scaled = sum.mul_scalar(2)
  return scaled.open()
```

Opening a share reconstructs the value into the execution context. Use `open()` deliberately at the point where your program needs a clear result.

## Batch open

Use batch operations when a program naturally returns multiple shares:

```stoffel
def main() -> list[int64]:
  var left = Share.from_clear(40)
  var right = Share.from_clear(2)
  var values: list[Share] = [left, right, left.add(right)]
  return Share.batch_open(values)
```

## ClientStore builtins

`ClientStore` bridges external client input material into a Stoffel program.

| Builtin | Purpose |
| --- | --- |
| `ClientStore.get_number_clients()` | Total known client slots. |
| `ClientStore.get_number_input_clients()` | Clients with input material. |
| `ClientStore.get_number_output_clients()` | Output-capable clients. |
| `ClientStore.take_share(client_slot, input_index)` | Load an integer share from a client slot. |
| `ClientStore.take_share_fixed(client_slot, input_index)` | Load a fixed-point share from a client slot. |

Example:

```stoffel
def main() -> int64:
  var client_count = ClientStore.get_number_clients()
  var share = ClientStore.take_share(0, 0)
  var opened: int64 = share.open()
  return opened + client_count
```

CLI local execution:

```bash
stoffel run --local --client-input 0=42 --parties 5 --threshold 1
```

Rust SDK local execution:

```rust
use stoffel::prelude::*;

# async fn example() -> stoffel::Result<()> {
let result = Stoffel::compile(
    "def main() -> int64:\n  var share = ClientStore.take_share(0, 0)\n  return share.open()",
)?
.parties(5)
.threshold(1)
.with_client_input(0, &[42_i64])
.execute_local()
.await?;
# Ok(())
# }
```

## Mpc builtins

`Mpc` exposes runtime metadata and capability checks:

```text
Mpc.party_id
Mpc.n_parties
Mpc.threshold
Mpc.is_ready
Mpc.instance_id
Mpc.protocol_name
Mpc.curve
Mpc.field
Mpc.has_capability
Mpc.capabilities
Mpc.rand
Mpc.rand_int
```

Example:

```stoffel
def main() -> int64:
  var score: int64 = Mpc.party_id()
  score = score + Mpc.n_parties()
  score = score + Mpc.threshold()

  if Mpc.is_ready():
    score = score + 10

  if Mpc.has_capability("multiplication"):
    score = score + 100

  return score
```

## MpcOutput builtins

Use `MpcOutput.send_to_client` when a program should deliver share outputs to client slots through the coordinator/output path:

```stoffel
def main() -> int64:
  var client_count = ClientStore.get_number_clients()
  var result = ClientStore.take_share(0, 0).add_scalar(5)

  if Mpc.has_capability("client-output"):
    var outputs: list[Share] = []
    outputs.append(result)
    MpcOutput.send_to_client(0, outputs)

  return client_count
```

`Share.send_to_client(client_slot)` is also available for single-share output flows.

## Crypto, Bytes, Rbc, and Avss modules

The VM also includes module-style builtins for cryptographic helpers, byte-array operations, reliable broadcast helpers, and AVSS-specific operations. These are lower-level integration surfaces; prefer checked examples in `crates/stoffel-lang/examples/` for production-facing patterns.
