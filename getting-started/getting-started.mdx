Start with the `stoffel` CLI. The CLI creates projects, checks StoffelLang source, builds `.stflb` bytecode, and runs local MPC development workflows.

## Recommended path

1. [Install the `stoffel` CLI](./installation).
2. [Create and run a project](./quick-start).
3. [Learn the daily CLI commands](./basic-usage).
4. [Build a small secret-shared example](./first-project).

## What you will use

- `stoffel init` to create a project.
- `src/main.stfl` for the StoffelLang program.
- `Stoffel.toml` for package, build, and MPC topology settings.
- `stoffel check` for fast validation.
- `stoffel build` to produce `.stflb` bytecode.
- `stoffel run` and `stoffel dev` for local MPC development.
- `src/main.rs` when you want the Rust SDK wrapper to call the same program from application code.

## Local MPC in the getting-started flow

Local MPC testing runs several MPC nodes/processes locally on your machine. Use it to check that private inputs, share operations, openings, and client-directed outputs behave as expected before moving to network or deployment work.

For CLI workflows, use:

```bash
stoffel run
stoffel dev --once
```

For the Rust SDK wrapper, use `.execute_local().await?`. That call runs the compiled program against a local MPC test network using the parties and threshold configured by the SDK builder or project settings.

## What not to install separately

Do not install lower-level runtime crates as part of the normal getting-started path. App developers should start with the `stoffel` CLI and the Rust SDK wrapper generated by the default project template.
