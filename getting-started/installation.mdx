# Installation

Install Stoffel first, then choose the build path your app needs. The installer gives you the `stoffel` CLI. The [`Stoffel-Labs/stoffel`](https://github.com/Stoffel-Labs/stoffel) repository gives you the Rust SDK source, examples, and local MPC runner.

<Note>
**Stoffel 0.1.0 docs**

Start with the CLI installer. Clone `Stoffel-Labs/stoffel` when your app needs SDK source, examples, or a local MPC runner.
</Note>

## Prerequisites

- A Unix-like shell: Linux, macOS, or Windows with WSL2
- `curl` or `wget`
- Rust and Cargo when your app uses the Rust SDK or installs the local MPC runner from crates.io
- Git only when you need the source repository, examples, or workspace development
- 4 GB RAM minimum; 8 GB recommended for local MPC development

## Install the CLI

```bash
curl -fsSL https://get.stoffelmpc.com | sh
```

The installer downloads the current `stoffel` binary, verifies its checksum, and installs it to `~/.local/bin` by default. Add that directory to your shell path if needed:

```bash
export PATH="$HOME/.local/bin:$PATH"
stoffel --help
```

You should see the project commands you use in the normal dev loop: `init`, `check`, `compile`, `build`, `run`, `dev`, `test`, `status`, `clean`, and `update`.

Pin a version or install somewhere else when you need a reproducible toolchain:

```bash
curl -fsSL https://get.stoffelmpc.com | sh -s -- --version 0.1.0
curl -fsSL https://get.stoffelmpc.com | STOFFEL_INSTALL_DIR="$HOME/bin" sh
```

## Add Rust crates from crates.io

Install Rust if needed:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
rustc --version
cargo --version
```

Add the Rust SDK to your app from crates.io:

```toml
[dependencies]
stoffel = { package = "stoffel-rust-sdk", version = "0.1.0" }
tokio = { version = "1.47.1", features = ["macros", "rt-multi-thread"] }
```

Other published crates are available when your app needs lower-level surfaces:

| Crate | Use when |
| --- | --- |
| `stoffel-rust-sdk` | Build app integrations that compile, load, and run Stoffel programs. |
| `stoffel-vm-runner` | Install the `stoffel-run` local MPC runner. |
| `stoffellang` | Use the Stoffel Lang compiler from Rust. |
| `stoffel-vm` | Embed the VM runtime directly. |
| `stoffel-vm-types` | Work with bytecode, instructions, and runtime value types. |

Install the local MPC runner from crates.io when your app needs local MPC execution:

```bash
cargo install stoffel-vm-runner
stoffel-run --help
```

## Optional: build from source

Use the source checkout when your product needs repository examples, workspace development, or local changes to the SDK/VM crates:

```bash
git clone https://github.com/Stoffel-Labs/stoffel.git
cd stoffel
cargo build
cargo build -p stoffel-vm --bin stoffel-run
```

The CLI install and crates.io packages above are enough for the normal app dev loop.

## Local runner path

If you installed `stoffel-vm-runner` from crates.io, the runner is on your Cargo bin path as `stoffel-run`. If you built from source, the debug runner is at:

```text
stoffel/target/debug/stoffel-run
```

Pass the runner path when a command or SDK example needs it:

```bash
stoffel dev \
  --runner "$HOME/.cargo/bin/stoffel-run" \
  --parties 5 \
  --threshold 1 \
  --once
```

If you installed or built Stoffel in another directory, replace the path with your local checkout path.

## Optional: release runner build

For faster local MPC execution from a source checkout, build the runner in release mode:

```bash
cargo build --release -p stoffel-vm --bin stoffel-run
```

The release runner path is then:

```text
stoffel/target/release/stoffel-run
```

## Add the Rust SDK to an application

The Rust SDK package on crates.io is `stoffel-rust-sdk`. Import it in Rust code as `stoffel` by renaming the package in `Cargo.toml`:

```toml
[dependencies]
stoffel = { package = "stoffel-rust-sdk", version = "0.1.0" }
tokio = { version = "1.47.1", features = ["macros", "rt-multi-thread"] }
```

When developing against a local checkout, use a path dependency:

```toml
[dependencies]
stoffel = { package = "stoffel-rust-sdk", path = "../stoffel/crates/stoffel-rust-sdk" }
tokio = { version = "1.47.1", features = ["macros", "rt-multi-thread"] }
```

<Note>
Use the CLI installer for project commands. Use crates.io for app dependencies, and use a repository path only when your Rust app needs local SDK source changes.
</Note>

## Create a test project

```bash
stoffel init hello-mpc
cd hello-mpc
stoffel check
stoffel build
stoffel run
```

If your template program does not use named function inputs, use the inputs shown in that project's generated `README.md` or `src/main.stfl` instead.

## Troubleshooting

### `stoffel` is not found

Make sure Cargo's bin directory is on your path:

```bash
export PATH="$HOME/.cargo/bin:$PATH"
```

Add that line to your shell profile if needed.

### Local MPC says `stoffel-run` is missing

Install the runner from crates.io or build it from source, then pass its path:

```bash
cargo install stoffel-vm-runner
stoffel dev --runner "$HOME/.cargo/bin/stoffel-run" --once
```

### Cargo builds take a while

The SDK and runner pull MPC, networking, and coordinator crates through Cargo. The first build may take several minutes while Cargo fetches and compiles dependencies.

### Native Windows issues

Use WSL2 for the 0.1.0 workflow unless you are specifically testing native Windows support.
