# Ecosystem Overview

Stoffel 0.1.0 centers on [`Stoffel-Labs/stoffel`](https://github.com/Stoffel-Labs/stoffel), the repository for the CLI, compiler, Rust SDK, VM runtime, and shared bytecode/type definitions.

Stoffel gives you a direct path to private computation: write the workflow, mark sensitive values explicitly, run clear checks locally, then exercise secret-shared execution through the CLI or Rust SDK.

## Components in 0.1.0

| Component | Location | Developer guidance |
| --- | --- | --- |
| Stoffel repository | [`Stoffel-Labs/stoffel`](https://github.com/Stoffel-Labs/stoffel) | Start here. It contains the CLI, compiler, Rust SDK, VM runtime, bytecode/types crates, runtime helper, C FFI, and examples. |
| Stoffel CLI | `crates/stoffel-cli` / binary `stoffel` | Cargo-like project workflow for `init`, `check`, `compile`, `build`, `run`, `dev`, and tests. |
| Rust SDK | `stoffel-rust-sdk` on crates.io / crate `stoffel` | Primary application API for compile/load/run, local clear checks, local MPC, bytecode helpers, and network configuration surfaces. |
| Stoffel VM runtime | `stoffel-vm` on crates.io / `crates/stoffel-vm` | Register-based runtime with clear values, secret shares, builtins, hooks, FFI, and the `stoffel-run` helper. Most app developers reach it through the CLI or SDK. |
| VM types and bytecode | `stoffel-vm-types` on crates.io / `crates/stoffel-vm-types` | Shared instruction, value, function, and `.stflb` compiled binary types. |
| Stoffel Lang compiler crate | `stoffellang` on crates.io / `crates/stoffel-lang` | Compiler surface for Rust tools that need to compile Stoffel Lang directly. |
| Runner crate | `stoffel-vm-runner` on crates.io / `crates/stoffel-vm-runner` | Installs the `stoffel-run` helper for local MPC and VM execution flows. |
| MPC/network dependencies | `mpc-protocols`, `stoffel-networking`, `stoffel-mpc-coordinator` | Pulled by Cargo for protocol, networking, and coordination support; app developers normally use the CLI and SDK first. |

## Security review scope

`mpc-protocols` has received an external audit from Zellic. Other parts of the codebase have had AI-assisted security review, including tools such as veria.dev. Treat audit and review coverage as component-scoped. Validate the full application, integration code, configuration, and threat model your product relies on.

## Primary developer workflow

### 1. Install the tools

Install the CLI:

```bash
curl -fsSL https://get.stoffelmpc.com | sh
export PATH="$HOME/.local/bin:$PATH"
stoffel --help
```

If your app needs local MPC runner flows, install `stoffel-run` from crates.io:

```bash
cargo install stoffel-vm-runner --version 0.1.0 --locked
stoffel-run --help
```

For Stoffel 0.1.0, `stoffel-run` is the local MPC helper used by SDK and CLI execution paths.

### 2. Create a project

```bash
stoffel init hello-mpc
cd hello-mpc
```

The default template currently creates a hybrid Rust/Stoffel project:

```text
hello-mpc/
в”ңв”Җв”Җ Cargo.toml
в”ңв”Җв”Җ README.md
в”ңв”Җв”Җ Stoffel.toml
в””в”Җв”Җ src/
    в”ңв”Җв”Җ main.rs
    в”ңв”Җв”Җ main.stfl
    в””в”Җв”Җ stoffel_bindings.rs
```

### 3. Compile, inspect, and run

```bash
stoffel check
stoffel build
stoffel compile --disassemble target/debug/hello-mpc.stflb
stoffel run
```

For local MPC development:

```bash
stoffel dev \
  --runner $HOME/.cargo/bin/stoffel-run \
  --parties 5 \
  --threshold 1 \
  --once

```

Use `stoffel dev --once` for a script-friendly one-shot run.

## Component details

### Stoffel CLI

The `stoffel` binary is a Cargo-like command-line tool. The 0.1.0 CLI commands are:

- `init` / `new`: create projects from templates.
- `check`: validate source and MPC settings without writing bytecode.
- `compile`: compile source or projects to `.stflb`, or disassemble bytecode.
- `build`: write project bytecode under `target/`.
- `run`: run a project, `.stfl` source file, or `.stflb` bytecode file.
- `dev`: watch and rerun local MPC flows.
- `test`: run no-argument Stoffel test functions.
- `status` / `doctor`: show project health and environment status.
- `clean`: remove generated artifacts.
- `update` / `upgrade`: check or update CLI/project dependencies.

### Stoffel Lang compiler

Stoffel Lang is the source language for the VM. The 0.1.0 syntax is Python-style:

```stoffel
def main(a: int64, b: int64) -> int64:
  return a + b
```

Secret-shared values are explicit in the 0.1.0 local MPC smoke path:

```stoffel
def add_private_values(a: Share, b: Share) -> int64:
  var sum = Share.add(a, b)
  return sum.open()
```

### Rust SDK

The Rust SDK is the primary application API for 0.1.0. It exposes `Stoffel::compile`, `Stoffel::load`, `with_inputs`, `with_client_input`, `execute_clear`, `execute_local`, bytecode save/load helpers, network deployment builders, and observability helpers.

```rust
use stoffel::prelude::*;

fn main() -> stoffel::Result<()> {
    let result = Stoffel::compile("def main(a: int64, b: int64) -> int64:\n  return a + b")?
        .with_inputs(&[("a", 42_i64), ("b", 58_i64)])
        .execute_clear()?;

    println!("Result: {}", result[0]);
    Ok(())
}
```

### Stoffel VM

The VM is a register-based runtime with clear values, secret shares, objects, arrays, closures, FFI support, VM hooks, bytecode support, and MPC-focused builtins such as `Share.*`, `Mpc.*`, `ClientStore.*`, and `MpcOutput.*`.

### MPC/network backends

HoneyBadger MPC, AVSS, reliable broadcast, and QUIC networking integrations are wired into the repository through Rust dependencies. Local developer workflows normally use the SDK/CLI and `stoffel-run` rather than integrating those crates directly.
