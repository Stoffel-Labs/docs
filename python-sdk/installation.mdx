# Installation

<Note>
**Python integration starts with the CLI template.**

Use the `stoffel` repository and the CLI Python template for Python-oriented project scaffolding. Use the Rust SDK for verified application execution.
</Note>

## Prerequisites

Install the 0.1.0 CLI from the matching `stoffel` source checkout first:

```bash
git clone https://github.com/Stoffel-Labs/stoffel.git
cd stoffel
git checkout dev
cargo install --path crates/stoffel-cli
cargo build -p stoffel-vm --bin stoffel-run
```

Verify the Python template is available:

```bash
stoffel init --help
```

The help output should list `python (py)` among the supported templates.

## Create a Python scaffold

```bash
stoffel init my-python-project --template python
cd my-python-project
```

Then inspect the generated README and manifests:

```bash
ls
cat README.md
find stoffel -maxdepth 3 -type f
```

Run the nested Stoffel program with the CLI:

```bash
stoffel check stoffel
stoffel run stoffel --input a=40 --input b=2
```

If your generated program expects different inputs, follow that template's `README.md` or `stoffel/src/program.stfl`.

## Python package status

Stoffel 0.1.0 uses the CLI Python template rather than a Python SDK package. The documented install path is the repository CLI plus the Rust SDK.
